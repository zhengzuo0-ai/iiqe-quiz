# IIQE 泉泉专用学习网站 — Project Context

## 项目概述

为Sherry（泉泉）备考香港保险中介人资格考试（IIQE）搭建的AI驱动刷题网站。考试日期：**2026年4月8日**，同一天连考两场：
- 卷三（长期保险）14:00-15:15，50题，75分钟，及格70%（35/50）
- 卷一（保险原理及实务）15:45-17:45，75题，120分钟，及格70%（53/75）

## 当前状态

已有一个React单文件prototype（`iiqe-quiz.jsx`），核心功能可用：
- 卷一（7章）和卷三（5章）按章节练习
- Claude API实时生成中文选择题 + 详细解释
- 模拟考试模式（计时、按权重分配章节）
- 本地localStorage存储做题进度
- 手机友好的响应式界面

## 需要解决的关键问题

### 1. 出题速度太慢（最高优先级）

当前每道题都实时调Claude API，latency约3-5秒，体验很差。解决方案（按推荐优先级）：

**方案A：预生成题库 + 缓存**
- 用脚本批量调API预生成每章50-100道题，存为JSON文件或数据库
- 前端直接从题库随机抽题，毫秒级响应
- 定期补充新题防止刷完
- 这是最优方案：零延迟、离线可用、成本可控

**方案B：流式响应（Streaming）**
- 用Anthropic streaming API，题目逐字显示
- 体感快很多但实际总时间不变
- 适合用在"AI讲解"等需要实时生成的场景

**方案C：预加载下一题**
- 用户答当前题时，后台预取下一题
- 简单有效，但首题仍慢

**建议组合：题库用方案A，答错后的"深度讲解/追问"用方案B streaming。**

### 2. 查漏补缺 / 错题记忆系统

当前只记录了每章正确率，需要升级为完整的错题系统：

**数据模型：**
```
ErrorRecord {
  id: string
  paper: "paper1" | "paper3"
  chapterId: string
  question: object          // 完整题目
  userAnswer: string        // 用户选的
  correctAnswer: string     // 正确答案
  timestamp: number
  reviewCount: number       // 复习次数
  lastReviewAt: number      // 上次复习时间
  mastered: boolean         // 是否已掌握
}
```

**功能需求：**
- 错题本页面：按章节分组查看所有做错的题
- 错题重做：从错题库中随机抽题再练
- 间隔复习：基于艾宾浩斯遗忘曲线，做错的题在1天、3天、7天后自动推送复习
- 薄弱分析看板：哪些考点反复出错，可视化展示
- 导出错题报告（可选）

### 3. 部署上线

需要让Sherry通过一个URL直接访问。选项：

**方案A：Vercel/Netlify 静态部署（推荐）**
- 最简单，fork到GitHub然后一键deploy
- 免费tier足够
- 需要把项目从单文件改为标准React项目结构

**方案B：Cloudflare Pages**
- 同样免费、全球CDN
- 适合静态站

**方案C：自有服务器**
- 如果需要后端（比如题库API、用户数据持久化），可以加一个轻量后端
- FastAPI + SQLite 或 Supabase

**建议架构：Vercel前端 + Supabase后端（免费tier，提供数据库和auth）**

## AI出题Prompt

当前使用的prompt模板（已验证效果良好，生成的题目质量接近真题）：

```
你是一位香港保险中介人资格考试（IIQE）{卷名}的出题专家。

请根据以下章节生成一道高质量的中文选择题：

章节：{章节名}
考点范围：{具体考点}

要求：
1. 题目和选项全部用中文
2. 提供4个选项 A、B、C、D
3. 贴近真实考试风格，注重理解而非死记硬背
4. 适当使用情景题
5. 错误选项要有迷惑性
6. 解释要详细，帮助考生真正理解

JSON格式输出：question, options, correct, explanation, key_concept
```

**批量生成脚本的prompt需要额外加：**
- 去重指令：附上已生成的题目摘要，避免重复
- 难度分级：easy/medium/hard各占比约20%/60%/20%
- 考点覆盖：确保每个子考点都有题目覆盖
